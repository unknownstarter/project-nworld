# Test-Driven Development

RED → GREEN → REFACTOR 사이클을 엄격하게 따른다.

## Cycle

### 1. RED (실패하는 테스트 작성)
- 구현하려는 기능의 **최소 단위** 테스트를 작성
- 테스트를 실행하여 **실패**하는 것을 확인
- 테스트가 올바른 이유로 실패하는지 확인

### 2. GREEN (최소한의 코드로 통과)
- 테스트를 통과시키는 **가장 단순한** 코드 작성
- 하드코딩도 괜찮음 (다음 테스트가 일반화를 강제)
- IMPORTANT: 이 단계에서 리팩토링하지 않는다

### 3. REFACTOR (코드 개선)
- 모든 테스트가 통과한 상태에서 코드 개선
- 중복 제거, 명확한 네이밍
- 테스트가 여전히 통과하는지 확인

## Rules
1. 실패하는 테스트 없이 프로덕션 코드를 작성하지 않는다
2. 실패를 보여주기에 충분한 만큼만 테스트를 작성한다
3. 테스트를 통과시키기에 충분한 만큼만 프로덕션 코드를 작성한다
4. Mock은 최소화한다 (외부 의존성만 Mock)
