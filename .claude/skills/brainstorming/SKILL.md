# Brainstorming

소크라테스식 질문법으로 아이디어를 발전시킨다.

## Process

### 1. 발산 (Diverge)
- 아이디어를 제한 없이 나열 (양 > 질)
- "Yes, and..." 원칙 (부정하지 않고 확장)
- 반대 입장에서도 생각 (Reverse brainstorming)
- 다른 산업/분야에서 유추 (Analogical thinking)

### 2. 소크라테스 검증
아이디어마다 다음 질문을 적용:
- **근본**: 이것이 정말 해결하려는 문제는 무엇인가?
- **가정**: 어떤 가정 위에 서 있는가? 그 가정이 틀리면?
- **증거**: 이것이 작동할 거라는 근거는 무엇인가?
- **대안**: 이것 말고 같은 문제를 푸는 다른 방법은?
- **결과**: 이것이 성공하면 어떤 2차 효과가 발생하는가?
- **반론**: 이것에 반대하는 사람은 뭐라고 할 것인가?

### 3. 수렴 (Converge)
- Impact/Effort 매트릭스로 분류
- 상위 3개 아이디어에 집중
- 각 아이디어의 핵심 가설 1개 추출
- 검증 실험 설계

## Output Format
```
## Idea: [제목]
### 핵심 가설
[한 문장으로 표현된 검증 가능한 가설]
### 검증 방법
[가장 빠르게 가설을 검증하는 방법]
### 리스크
[가장 큰 리스크 1-2개]
### 잠재력
[성공 시 임팩트]
```
