# User Interview Design

## Interview Framework

### 1. 준비
- 연구 질문(RQ) 정의
- 가설 목록 (검증하려는 것)
- 참가자 기준 정의 (스크리너)
- 인터뷰 가이드 작성 (45-60분)

### 2. 질문 설계 원칙
- IMPORTANT: 유도 질문 금지 ("이 기능 좋지 않나요?" → X)
- 과거 행동에 대해 묻기 ("마지막으로 X를 한 게 언제예요?")
- "왜?"보다 "어떻게?" ("그때 어떻게 하셨어요?")
- 가설 직접 물어보지 않기 (간접적으로 탐색)
- 침묵을 견디기 (5초 이상 기다리기)

### 3. 인터뷰 구조
```
[5분]  Warm-up: 자기소개, 라포 형성
[10분] Context: 일상, 현재 행동 패턴
[20분] Deep dive: 핵심 주제 탐색
[10분] Concept testing: 아이디어/프로토타입 반응
[5분]  Wrap-up: 추가 생각, 추천인 요청
```

### 4. 분석
- Affinity mapping: 유사 응답 그룹핑
- Pattern recognition: 3명 이상 반복된 패턴
- Surprise finding: 예상과 다른 발견
- 행동 vs 말의 차이 포착
