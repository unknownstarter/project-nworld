# Deep Research

출처: nateberkopec/deep-research + 업계 패턴

## Multi-Phase Research Process

### Phase 1: 범위 정의 (2분)
```
1. 연구 질문을 3-5개 하위 질문으로 분해
2. 각 하위 질문에 대한 검색 전략 수립
3. 필요한 소스 유형 식별 (학술, 뉴스, 데이터, 전문가 의견)
```

### Phase 2: 정보 수집 (병렬)
```
SubAgent 1: 질문 1 → 웹 검색 → 콘텐츠 분석 → 요약
SubAgent 2: 질문 2 → 웹 검색 → 콘텐츠 분석 → 요약
...
SubAgent N: 질문 N → 웹 검색 → 콘텐츠 분석 → 요약
```

각 서브에이전트가 수행하는 작업:
1. 키워드 검색 (3-5개 쿼리 변형)
2. 상위 결과 페이지 방문 & 콘텐츠 추출
3. 핵심 사실, 통계, 인용 수집
4. 출처 URL과 신뢰도 기록

### Phase 3: 종합 & 분석 (5분)
```
1. 서브에이전트 결과 통합
2. 중복 제거 & 모순 식별
3. 삼각 검증 (3개 이상 독립 소스로 확인)
4. 인사이트 도출 & 패턴 인식
5. 갭 분석 (아직 답변되지 않은 질문)
```

### Phase 4: 보고서 작성
```markdown
# [리서치 제목]

## Executive Summary
[30초 안에 읽을 수 있는 핵심 발견 3-5개]

## 연구 질문
1. [RQ1]
2. [RQ2]
...

## Findings

### Finding 1: [제목]
[근거, 데이터, 소스 인용]

> "직접 인용" — 출처

### Finding 2: ...

## 분석 & 시사점
[패턴, 트렌드, 우리에게 의미하는 바]

## 한계점
[이 리서치의 제한사항]

## Sources
1. [제목](URL) - 접근 일자
2. ...
```

## Quality Standards
- IMPORTANT: 모든 주장에 출처 명시 (URL 포함)
- 1차 자료 우선 (원본 보고서, 데이터셋, 공식 발표)
- 최신성 확인 (특히 AI/기술 분야: 6개월 이내)
- 편향 식별 및 명시
- 반대 의견도 공정하게 다룸
