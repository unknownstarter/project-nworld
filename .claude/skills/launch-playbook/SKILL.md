# Launch Playbook

출처: coreyhaines31/marketingskills (launch-strategy, pricing-strategy) + 업계 베스트 프랙티스

## Launch Tiers

### Tier 1: Soft Launch (소규모)
- 대상: 베타 사용자 50-200명
- 목적: PMF 검증, 크리티컬 버그 발견
- 채널: 직접 초대, DM, 이메일

### Tier 2: Community Launch
- 대상: 얼리어답터 커뮤니티
- 채널: Product Hunt, Hacker News, Reddit, X
- 목적: 초기 트랙션, 피드백, 소셜 프루프

### Tier 3: Public Launch
- 대상: 전체 시장
- 채널: PR, 광고, 파트너십, 인플루언서
- 목적: 대규모 성장

## Pre-Launch Checklist (D-30 to D-1)

### D-30: Foundation
- [ ] Landing page + 대기자 명단
- [ ] Brand story / Founding story 작성
- [ ] Press kit (로고, 스크린샷, 설명, 창업자 사진)
- [ ] 데모 비디오 제작

### D-14: Amplification
- [ ] 얼리어답터 50명 사전 접근 & 피드백
- [ ] 인플루언서/미디어 사전 접촉 (엠바고 제안)
- [ ] Product Hunt 헌터 섭외
- [ ] 소셜 미디어 티저 시작

### D-3: Final Prep
- [ ] 론칭일 실행 체크리스트 확정
- [ ] 팀별 역할 분담
- [ ] 모니터링 대시보드 준비
- [ ] 고객 지원 채널 오픈

## Launch Day (D-Day)

### Timeline
```
06:00  Product Hunt 제출 (PST 자정)
07:00  블로그 포스트 발행
08:00  X/Twitter thread 발사
09:00  LinkedIn 포스트
10:00  Hacker News 제출
12:00  이메일 뉴스레터 발송
       (이후: 댓글 응답, 커뮤니티 참여, 실시간 대응)
```

### Key Assets
1. **Product Hunt**: 태그라인 (60자), 설명, 4-6 이미지, 비디오
2. **Blog Post**: 창업 스토리 + 제품 소개 + 비전
3. **Twitter Thread**: 8-12트윗, 스크린샷 포함
4. **Demo Video**: 2분 이내

## Pricing Strategy

### Model Options
| 모델 | 적합한 경우 | 예시 |
|------|-------------|------|
| Freemium | 바이럴 + 네트워크 효과 | Notion, Figma |
| Free Trial | B2B, 높은 전환 가치 | Salesforce |
| Usage-based | API, 인프라 | AWS, OpenAI |
| Flat Rate | 단순함이 경쟁력 | Basecamp |
| Per Seat | 팀 기반 | Slack |

### Pricing Psychology
- 앵커링: 비싼 플랜을 먼저 보여주기
- 3-tier: Good / Better / Best (가운데 강조)
- 연간 결제 20-30% 할인
- 무료 티어: 충분히 가치있되 업그레이드 동기 부여
- "$X/month" (월 표시가 더 작아 보임)

## Post-Launch (D+1 to D+30)
- [ ] 사용자 피드백 수집 & 분류

- [ ] 이탈 사용자 인터뷰 (5-10명)
- [ ] Quick wins 빠르게 반영
- [ ] 회고 진행
